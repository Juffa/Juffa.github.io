Third post, misadventures in javascript

Read

I've been banging my head into javascript, and finally I thought I got it right.. but turns out I didn't. The clever ajax script that was supposed to read file names from the server, so that I wouldn't have to "know" what or how many files there are... It doesn't work online. It does work, but only in my test environment. Isn't it fun when security features block your "clever" scripts? I dont want to have to adjust the file numbers into index.html when I add another "blog post".
Could I brute-force it? Like, test if file number 20 exists, if not, then test file number 10, if yes, then file 15, if not then 13 etc etc. And then load the elements from the files. A friend of mine suggested writing a script for uploading the pages, and that script would modify the values. I may have to look into that. For now I'll just adjust the number by hand.

For now I need more "blog posts" to test the code. Every post has identical id's in elements and they need to be changed when they are loaded. Of course this could be changed also by the script that would upload the pages.

The script that "almost" worked was like this: 

                      
    
      pages=[];
      $.when(
        $.ajax({
          url:"/posts/",
          success: function(data){
            $(data).find("a:contains(.html)").each(function(){
              pages.push($(this).attr("href"));
            });
          }
        })
      ).then(function(){
        $.each(pages,function(index,value){
          $("#container").prepend($("<div></div>").load(value +" .blog", function(){}));
        });
      });
